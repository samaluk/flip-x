## Cursor Cloud specific instructions

### Services overview

| Service | Command | Port | Notes |
|---------|---------|------|-------|
| Convex backend (local) | `HOME=/tmp/convex-home CONVEX_DEPLOYMENT= npx convex dev --local --typecheck disable --tail-logs disable` | 3210 | Interactive prompts appear on first run; select "Start without an account" then confirm with `Y`. Creates `.env.local` with local URLs automatically. |
| Next.js dev server | `pnpm dev` | 3000 | Reads `NEXT_PUBLIC_CONVEX_URL` from `.env.local`. |

### Running services

1. Start Convex local backend first â€” it writes `.env.local` with `NEXT_PUBLIC_CONVEX_URL=http://127.0.0.1:3210`.
2. Then start Next.js dev server with `pnpm dev`.
3. The Convex interactive setup prompts only appear once; subsequent runs auto-connect.

### Key commands

- **Lint:** `pnpm lint` (oxlint)
- **Unit tests:** `pnpm test` (vitest, jsdom)
- **Build:** `pnpm build`
- **Convex backend tests:** `pnpm test:convex` (needs local Convex on port 3210)

### Gotchas

- The `pnpm-workspace.yaml` lists `ignoredBuiltDependencies` for `sharp` and `unrs-resolver`. After `pnpm install`, build scripts for `esbuild`, `@swc/core`, `@parcel/watcher`, and `msw` are blocked by default. To allow them, temporarily add those packages to `onlyBuiltDependencies` in `pnpm-workspace.yaml` and re-run `pnpm install`, then revert the file. Alternatively, run `pnpm rebuild esbuild @swc/core @parcel/watcher msw`.
- The Convex CLI prompts interactively on first local setup. Use `CONVEX_AGENT_MODE=anonymous` if available, or pipe through non-interactive answers.
- `.env.local` is git-ignored and auto-generated by `npx convex dev --local`.
